## What's new

- **Explicit client integration on Windows.** `install.ps1` detects Claude Code
  and Codex CLI, including common npm and per-user installation paths, then asks
  whether to configure Claude, Codex, both, or neither. It no longer creates
  Claude settings when Claude Code is absent.
- **Predictable automation.** Non-interactive `-Yes` installs now require
  `-Integration Claude`, `Codex`, `Both`, or `None` (or `-CoreOnly`). This keeps
  unattended setup from silently changing multiple client configurations.
- **Core-only and configure-later workflows.** `-CoreOnly` installs Ariadne,
  Qdrant, Ollama, models, and the tray without touching an AI client.
  `-ConfigureClients -Integration ...` connects an already installed core after
  Claude Code or Codex CLI is installed.
- **Update isolation.** Consent-gated tray updates preserve existing client
  configurations unchanged. Updating binaries never enrolls a newly detected
  client.
- **Physical and virtual-machine preflight.** The installer reports Windows,
  machine type, CPU, cores, RAM, and free disk. Default local models require at
  least 8 GB RAM and 15 GB free disk; 16 GB RAM and 4+ cores are recommended.
  Virtual machines get explicit vCPU and modern CPU-type guidance.
- **Direct Qdrant compatibility check.** Before creating the scheduled task, a
  fresh installation executes `qdrant.exe --version` and reports loader or CPU
  incompatibility immediately.

## Windows examples

Interactive install with detected-client selection:

```powershell
irm https://raw.githubusercontent.com/mclaut/ariadne/main/install.ps1 -OutFile install.ps1
.\install.ps1
```

Unattended installation:

```powershell
.\install.ps1 -Yes -Integration Claude
.\install.ps1 -Yes -Integration Codex
.\install.ps1 -Yes -Integration Both
.\install.ps1 -Yes -CoreOnly
```

Configure a client later without reinstalling the core:

```powershell
.\install.ps1 -ConfigureClients -Integration Claude
.\install.ps1 -ConfigureClients -Integration Codex
```

`-Integration None` and `-CoreOnly` do not remove existing settings; they simply
make no client changes. Tray updates likewise preserve integrations.

## Also included from v0.4.0

Ariadne reports local content-free estimates of represented, delivered, and net
avoided recall context through `ariadnectl metrics`, JSON status, and the tray.
Metrics never contain memory or transcript text.

## Publication and rollout

- Release: [Ariadne v0.5.0](https://github.com/mclaut/ariadne/releases/tag/v0.5.0)
- Announcement: [GitHub Discussion #3](https://github.com/mclaut/ariadne/discussions/3)
- Project site: [mclaut.github.io/ariadne](https://mclaut.github.io/ariadne/)
- The release published five platform archives, an MCPB bundle, a CycloneDX
  SBOM, `server.json`, SHA-256 checksums, and a keyless Sigstore bundle.
- The official MCP Registry accepted the v0.5.0 metadata publication.
- Release archives were verified against `SHA256SUMS` and rolled out on macOS
  arm64 plus Linux amd64 desktop and headless installations. Qdrant remained
  green, Ollama remained reachable, existing memory data was preserved, and
  the tray was restarted only on desktop installations.

The Windows AST check caught a non-ASCII em dash in `install.ps1` before the
tag was created. Windows PowerShell 5.1 can read UTF-8 scripts without a BOM
through a legacy code page, which made the parser report misleading errors at
later words such as `cores` and `RAM`. The installer is now ASCII-safe and the
Windows parser check remains a release gate.
