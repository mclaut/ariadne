## What's new

- **Automatic daily diary distillation.** Ariadne now consolidates session diary
  entries older than 24 hours into concise, durable `decisions`, `gotchas`, and
  `reference` memories grouped by project and day.
- **Stronger signal, smaller context.** The local model is instructed to retain
  rationale, verified root causes and fixes, critical constraints, and important
  open risks while discarding chronology, routine progress, duplicates, code,
  logs, and facts recoverable from the repository.
- **Fail-safe replacement.** Every real consolidation creates a Qdrant snapshot
  first. Distilled memories are saved before source diary entries are deleted;
  a failed group remains available for retry.
- **Safe preview.** `ariadnectl consolidate --before 24h --dry-run` shows exactly
  what would be retained without modifying the store.
- **Local-first privacy.** Consolidation uses the existing local Ollama summary
  model. Remote endpoints stay blocked unless the user explicitly enables them.
- **Daily maintenance.** macOS launchd and Linux systemd jobs now run both
  memfile synchronization and diary consolidation at 04:30, logging to
  `~/.ariadne/logs/maintenance.log`.

## Usage

```bash
# Preview diary entries older than one day
ariadnectl consolidate --before 24h --dry-run

# Back up, distill, save durable memories, then retire source diary
ariadnectl consolidate --before 24h

# Keep a longer review window
ariadnectl consolidate --before 48h
```

SessionEnd capture remains unchanged: raw transcripts are never stored, trivial
sessions are skipped, and each substantive session produces at most one curated
diary memory. Daily distillation adds a second retention layer so recurring
recall returns conclusions instead of session chronology.
