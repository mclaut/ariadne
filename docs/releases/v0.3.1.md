## What's fixed

- **Qdrant starts on clean Windows installations.** The official Qdrant
  `qdrant-x86_64-pc-windows-msvc.zip` binary depends on
  `VCRUNTIME140.dll`. Ariadne now detects Microsoft Visual C++ Runtime 14.44
  or newer before starting Qdrant. If needed, it downloads Microsoft's official
  x64 redistributable, verifies the Microsoft Authenticode signer, and asks for
  administrator approval for this prerequisite only.
- **Startup failures explain themselves.** Qdrant now gets 60 seconds to become
  healthy. PowerShell loader errors and nonzero exits are written to
  `~/.ariadne/logs/qdrant.log`; on failure, the installer prints the scheduled
  task result and the last 40 log lines.
- **Windows CI starts the real database.** The Windows job downloads the exact
  Qdrant version pinned by `install.ps1`, verifies its SHA-256 digest, launches
  it with Ariadne's loopback-only storage settings, and requires `/healthz` to
  pass.

## Install or repair on Windows

Run PowerShell as your regular user:

```powershell
irm https://raw.githubusercontent.com/mclaut/ariadne/main/install.ps1 -OutFile install.ps1
powershell -ExecutionPolicy Bypass -File .\install.ps1
```

If the Microsoft runtime is missing or outdated, Windows displays one
administrator-approval prompt for the Microsoft-signed prerequisite. Existing
Qdrant data under `~/.ariadne/qdrant-data` is preserved.
